Data

Data provided by Data SF

The original dataset contains 12 columns and over 21k rows. I used the filtering on the source website to make it much more compact.

The columns I've mainly used:

  1. Activity period: I used this column to calculate the month that each row describes.
  2. Number of passengers: This describes the amount of passengers flying in or out of SFO for a given activity period.
  3. Boarding area: This column is a text field specifying the boarding area that a flight used.

Wrangling

I did minor wrangling such as summing up the passenger counts by boarding area and separately by activity period.

I used the source website as well as Tableau and Excel for wrangling the data.

Visualizations

1. Bar Chart

Here the length of each bar corresponds to the number of emplaned passenger. I used one bar for each month. The text for the months is quite small but having many rows is useful for seeing the pattern in this one.

Here I've wrangled the data using the data source website interface and exce, I added up the passenger counts with the same activity period for each airline to calculate a monthly total.

One conclusion we can make looking at this visualization is that overall the amount of flights have increased over the years, however if we zoom in on years separately we can see a pattern emerge where there are more flights in the summer.

Data source: Data SF, Author: Nedim Azar

2. Pie Chart

Here the size of each slice corresponds to the proportion of boarding area used overall for the time period in question. Color choice was arbitrary.

To wrangle the data I used the data source website and Tableau, I added up the amount of passengers for each boarding area and limited the activity period.

We can observe a relatively equal distribution among boarding areas A-B-C-D-E-G but area F seems to have a much greater flux of passengers.

Data source: Data SF, Author: Nedim Azar
